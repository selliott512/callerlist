Introduction

CallerList was written by members of the Atheist Community of Austin in order to help with call screening for their live call in public access TV show "The Atheist Experience". Please see the cedits section for additional details

CallerList is a LAMP and AJAX based application that consists of one web interface for the "screener" (person who does caller data entry) and for the "crew" (people such as the host of the show who answer the calls on air).

The included callerlist.css file may be adjusted according to your preference.

See the "INSTALL" file for information on setting up this software.

Requirements

CallerList uses JavaScript extensively. Although a relatively new browser is required multiple browser vendors are supported. The following has been tested successfully.:

  • Chrome 4.0.249.30
  • Firefox 3.5.9
  • Internet Explorer 8
  • Nexus One 2.1-update1

Hopefully any version of PHP 5 works, but only PHP 5.2 has been tested.

Help

The host and screener pages are similar. For the most part the screener's page is a writable version of the host page with some additional features. This help begins by discussing some of the elements on each page and then ends with some information about key bindings for the screener.

Fields

Modified
This is the time that the screener last made some change to the database. Seeing this value can reassure the crew that the screener is actively working.
Time
This is the server time rendered with the client's time zone.
HTTP Status
The last status received in the HTTP response headers. 200 indicates success. A list of possible HTTP statuses can be found here. This field turns red when there is an err (when the status is not 200).
RTT
This is the Round Trip Time from the browser to the webserver. An increase in this value may indicate network congestion.
Pause
Don't retrieve new caller information unless the "Update" button is clicked explicitly.
Sync
When selected the client adjusts the delay between updates so that the updates are done on some multiple of the update rate. For example, if the default update rate of every 5 seconds is used then when this is selected updates will happen at 1:00:00, 1:00:05, ...
All (list page only)
When selected the crew can see the last 100 callers in chronological order regardless of online status or priority. The crew can use this when the want information on a recent past caller.
Flash (edit page only)
When the screener presses this button call clients turn red from 5 to 10 seconds. This is used when there is some urgent issue, such as the microphone being broken, where the screener needs to get the attention of the host quickly.
Manual Update
When pressed new caller information is retrieved rather than waiting 5 seconds.
Caller List
This is the list of callers. For the list page this is the online callers. For the edit page a fixed list of lines is displayed.
Message
This final field is used for the screener to send messages to the crew.

List Page

There are a few special key bindings for the list page.

P
Toggle the "Pause" checkbox.
S
Toggle the "Sync" checkbox.
A
Toggle the "All" checkbox.
U
Press the "Update" button.

Edit Page

It's helpful to understand what colors mean on the edit page. When a caller is placed online (the button in the online column) the online button turns yellow. When a field differs from it's original value it becomes a light blue. When The field is exited and the result is to to be sent to the server the field turns yellow until the server acknowledges the change. Consequently a buildup of yellow fields can indicate network trouble.

There are a few special key bindings for the edit page.

Delete
If the control key is pressed with delete the message field or row is cleared.
Down Arrow
When a name or topic field has focus this places the caller offline. The call screener does this when the host hangs up on a caller. If the control key is pressed with the down arrow the current row is also cleared.
Escape
When this is pressed edits to the current field are discarded.
Return
When this is pressed the changes to the current field are sent to the server. Usually the call screener presses either the up arrow or simple exits the field. If the control key is pressed with the return key all clients are flashed.
Up Arrow
When a name or topic field has focus this places the caller online. The call screener does this when queueing up a caller.
